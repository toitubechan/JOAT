## Card: What it is
image: loopsLogicHero
**Loops** let a program repeat actions, and **logic** (like "if this, then that") lets it make decisions. Together, they give programs the ability to do repetitive work tirelessly and respond differently to different situations. These two ideas are at the heart of what makes code powerful.

## Card: Why it matters
Loops and logic are core building blocks of nearly all programs — they're how code does useful, flexible work rather than just running once in a straight line. Understanding them is essential to learning programming, and the concepts map closely to everyday thinking, making them surprisingly intuitive.

## Card: Key terms
- **Loop**: code that repeats actions.
- **Logic / conditionals**: "if/then" decision-making in code.
- **Condition**: a true/false test that guides a decision.
- **Automation**: doing repetitive tasks automatically.

## Card: Loops: repeating tirelessly
A **loop** tells the program to repeat an action — perhaps a set number of times, or until a condition is met. Instead of writing the same instruction a thousand times, you write it once in a loop. This is a huge part of computing's power: computers excel at doing repetitive tasks endlessly and flawlessly (**automation**). Loops are how a program processes lists, repeats operations, or keeps running until something changes.

## Card: Logic: making decisions
image: decisionLogicHero
**Logic** (also called conditionals) lets a program make decisions based on a **condition** — a true/false test. The classic form is "**if** something is true, **then** do this (otherwise do that)." For example: *if* the password is correct, *then* log in; *otherwise* show an error. This is exactly like everyday decision-making, and it's what lets programs respond intelligently to different situations rather than always doing the same thing.

## Card: Common mistakes
Common misunderstandings: thinking programs only run straight through once (loops repeat and logic branches); finding loops and logic abstract (they map to everyday "repeat this" and "if this, then that" thinking); confusing the two; and underrating how powerful these simple ideas are combined. Loops (repeat) and logic (decide) are the core of flexible, powerful programs.

## Card: Quick recap
Loops let a program repeat actions tirelessly (great for automation and processing lists), while logic (conditionals) lets it make decisions — "if this is true, then do that." Together, they let programs do repetitive work and respond differently to different situations, rather than running once in a straight line. These two intuitive ideas are at the heart of what makes code powerful.
