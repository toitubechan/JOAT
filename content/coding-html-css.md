## Card: What it is
image: htmlCssHero
**HTML** and **CSS** are the two foundational languages of web pages. **HTML** provides the *structure and content* (headings, text, images, links), while **CSS** controls the *style and appearance* (colors, fonts, layout). Together they build the look of nearly every website. They're often the first things people learn in web development, and they're approachable for beginners.

## Card: Why it matters
HTML and CSS are the entry point to building for the web — accessible, visual, and immediately rewarding (you see results right away). Understanding their roles demystifies how web pages are made and is a great first step into coding. Even non-coders benefit from knowing what they are, given how central the web is.

## Card: Key terms
- **HTML**: the language defining a page's structure and content.
- **CSS**: the language controlling style and appearance.
- **Element**: a building block of HTML (like a heading or image).
- **Separation of concerns**: keeping structure and style separate.

## Card: HTML: structure and content
**HTML** (HyperText Markup Language) defines *what's on the page* and its structure: headings, paragraphs, images, links, buttons, and so on. Each piece is an **element**. Think of HTML as the skeleton and content of a page — it says "here's a heading, here's a paragraph, here's an image" — but on its own, it looks plain and unstyled. HTML answers *what* is on the page, not how it looks.

## Card: CSS: style and appearance
image: styleAppearanceHero
**CSS** (Cascading Style Sheets) controls *how the page looks* — colors, fonts, sizes, spacing, and layout. If HTML is the skeleton and content, CSS is the styling and design. The same HTML can look completely different depending on the CSS applied. Keeping structure (HTML) and style (CSS) separate — a "**separation of concerns**" — makes web pages easier to build and maintain. CSS answers *how* the page should appear.

## Card: Common mistakes
Common misunderstandings: thinking HTML and CSS are programming languages in the full sense (they structure and style, while languages like JavaScript add behavior); confusing their roles (HTML = structure/content, CSS = style); imagining one does both jobs; and finding them intimidating (they're among the most beginner-friendly, visual things to learn). HTML builds the page; CSS makes it look good.

## Card: Quick recap
HTML and CSS are the foundations of web pages: HTML provides the structure and content (headings, text, images — the skeleton), while CSS controls the style and appearance (colors, fonts, layout — the design). The same HTML looks different with different CSS. Keeping structure and style separate makes pages easier to build. They're beginner-friendly, visual, and a great first step into the web.
