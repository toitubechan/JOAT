## Card: What it is
image: variablesDataHero
In programming, a **variable** is a named container that stores a piece of information (**data**) — like a number, some text, or a true/false value. Programs constantly store, retrieve, and change data using variables. They're one of the most fundamental building blocks of code, and the idea is simple once you see it.

## Card: Why it matters
Almost every program works by storing and manipulating data, so variables are foundational to coding. Understanding them is one of the first and most important steps in learning to program. The concept is intuitive — like labeled boxes for information — and it unlocks understanding of how programs actually work with information.

## Card: Key terms
- **Variable**: a named container storing a piece of data.
- **Data**: information a program works with.
- **Data type**: the kind of data (number, text, true/false).
- **Value**: the actual content stored in a variable.

## Card: Labeled boxes for information
Think of a **variable** as a labeled box. You give it a name (the label) and put a **value** inside (the data) — for example, a variable named `score` holding the value `10`. The program can then use the name to read the value, or change what's stored (update `score` to `20`). This lets programs remember and work with information as they run. Variables are how code keeps track of things.

## Card: Different types of data
image: dataTypesHero
**Data** comes in different **data types**: numbers (for counting and math), text (often called "strings," like names or messages), and true/false values (called booleans, for yes/no logic), among others. Each type is used for different purposes. Knowing a value's type matters because you can do different things with it — you can add numbers, but adding text works differently. Programs juggle these data types constantly.

## Card: Common mistakes
Common misunderstandings: thinking variables are complicated (they're just labeled containers for data); confusing the variable's *name* with its *value*; not realizing data comes in different types used for different things; and being intimidated by the term. A variable is simply a named box holding information your program can use and change. It's a foundational, intuitive idea.

## Card: Quick recap
A variable is a named container that stores a piece of data — like a labeled box holding a value (a number, text, or true/false). Programs use variables to remember and work with information as they run, reading and updating values. Data comes in types (numbers, text, booleans) used for different purposes. Variables are a foundational, intuitive building block of code.
