## Card: What it is
image: versionControlHero
**Version control** is a system for tracking changes to code (or any files) over time, letting you save snapshots, undo mistakes, and collaborate with others. The most popular tool is **Git**. It's like a sophisticated "save history" and collaboration system, and it's essential to modern programming — even for solo coders.

## Card: Why it matters
Version control is a core part of how software is built today, both solo and in teams. Understanding the basic idea — tracking changes, saving snapshots, collaborating safely — demystifies an essential tool and explains how programmers avoid losing work and work together without chaos. It's a foundational skill for anyone serious about coding.

## Card: Key terms
- **Version control**: tracking changes to files over time.
- **Git**: the most popular version control tool.
- **Commit**: a saved snapshot of your work at a point in time.
- **Repository**: the project and its full history of changes.

## Card: A powerful save history
Imagine being able to save snapshots of your project at any point, see exactly what changed and when, and roll back to any earlier version if something breaks. That's **version control**. Each saved snapshot is a **commit**, and your project plus its full history lives in a **repository**. Unlike a single "save" that overwrites the previous version, version control keeps the whole timeline — so you can always go back, compare, or recover.

## Card: Collaboration without chaos
image: collaborationVersionHero
Version control's other superpower is **collaboration**. It lets multiple people work on the same project without overwriting each other's work — merging everyone's changes together and tracking who changed what. This is how teams of developers build software together without chaos. Tools like Git (and platforms built on it) make this possible, which is why version control is central to nearly all professional software development.

## Card: Common mistakes
Common misunderstandings: thinking version control is only for big teams (it's valuable even solo, as a powerful undo and backup); confusing it with simply saving a file (it keeps the whole history, not just the latest version); finding it intimidating (the core idea — snapshots and history — is simple); and not using it and losing work. Version control safely tracks changes and enables collaboration.

## Card: Quick recap
Version control tracks changes to files over time, letting you save snapshots (commits), undo mistakes, and roll back to earlier versions — your project and its full history live in a repository. It also enables collaboration, merging multiple people's work without overwriting. Git is the most popular tool. It's essential to modern coding, valuable even solo as a powerful undo and backup system.
