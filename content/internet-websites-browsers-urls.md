## Card: What it is
image: websiteBrowserUrlHero
A **website** is a collection of pages stored on a server. A **browser** (like Chrome, Safari, or Firefox) is the app that fetches and displays them. A **URL** is the web address you type to find a site. Together, these are the everyday tools you use to move around the web.

## Card: Why it matters
Understanding these basics helps you navigate confidently, spot dodgy or fake addresses, and recognize a secure connection. The difference between a safe site and a scam often hides in the URL — so reading web addresses is a genuine safety skill, not just trivia.

## Card: Key terms
- **Browser**: the app that displays web pages.
- **URL**: a web address, like www.example.com.
- **Domain**: the main name in a URL, like "example.com".
- **HTTPS**: a secure, encrypted version of a web connection.

## Card: Reading a URL
image: urlPartsHero
A **URL** has parts. Take `https://www.example.com/page`: `https` is the protocol, `example.com` is the **domain** (the site's core identity), and `/page` points to a specific page. The domain is the key bit for safety — it tells you *whose* site you're really on, no matter how official the rest looks.

## Card: HTTPS and the padlock
**HTTPS** (note the "s") means the connection is **encrypted** — data between you and the site is scrambled so others can't easily read it. Browsers often show a padlock for this. It's important for anything sensitive like logins or payments. But note: a padlock means the *connection* is secure, not that the *site* is trustworthy.

## Card: Common mistakes
A key mistake is trusting a site just because it has a padlock — scammers use HTTPS too; it only secures the connection, not the site's honesty. Another is not checking the **domain** carefully, since fakes use look-alikes (like "examp1e.com"). And many type sensitive info on sites reached via links rather than typing the address themselves.

## Card: Quick recap
A browser fetches websites stored on servers, found via URLs. The domain in a URL reveals whose site it really is — the key safety check. HTTPS and the padlock mean the connection is encrypted, important for logins and payments, but they don't prove the site itself is trustworthy. Read addresses carefully.
