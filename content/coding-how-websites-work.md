## Card: What it is
image: websitesWorkHero
A **website** is a collection of files stored on a server that your browser fetches and displays. Behind every web page is a system: your browser requests the page, a server sends it, and your browser turns the code into what you see. Understanding this demystifies the web and is the foundation for learning web development.

## Card: Why it matters
Websites are central to modern life, and understanding how they work helps you grasp the internet, troubleshoot issues, and take the first step toward building your own. It connects several tech concepts (servers, browsers, code) into a clear picture of what happens when you visit a page.

## Card: Key terms
- **Browser**: the app that requests and displays web pages.
- **Server**: a computer that stores and sends website files.
- **Front-end**: the part of a website you see and interact with.
- **Back-end**: the behind-the-scenes server logic and data.

## Card: Request and response
image: requestResponseHero
The basic flow: you enter a web address or click a link, and your **browser** sends a request over the internet to a **server** (a computer storing the website). The server responds by sending back the website's files. Your browser then reads those files and renders them into the page you see. This request-and-response between browser and server happens every time you load a page — usually in a fraction of a second.

## Card: Front-end and back-end
Websites have two sides. The **front-end** is what you see and interact with in the browser — the layout, text, images, and buttons (built with languages like HTML, CSS, and JavaScript). The **back-end** is the behind-the-scenes part on the server — handling logic, storing and retrieving data (like your account or a product list), and deciding what to send. A simple site may be mostly front-end; complex ones (with logins, databases) need a back-end too.

## Card: Common mistakes
Common misunderstandings: thinking a website is a single thing rather than files fetched from a server; not knowing the browser-server request/response flow; confusing front-end (what you see) with back-end (server-side logic); and imagining web pages just "exist" rather than being sent and rendered each visit. A website is files on a server, fetched by your browser and rendered into a page.

## Card: Quick recap
A website is files stored on a server. When you visit, your browser requests the page, the server sends the files back, and your browser renders them into what you see. Websites have a front-end (what you see and interact with) and often a back-end (server-side logic and data). This request-and-response happens every page load, demystifying the web and the basics of building for it.
