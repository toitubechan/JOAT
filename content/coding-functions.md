## Card: What it is
image: functionsHero
A **function** is a named, reusable block of code that performs a specific task. Instead of rewriting the same instructions repeatedly, you define a function once and "call" it whenever you need it. Functions are how programmers organize code into manageable, reusable pieces — a key idea for writing clear, efficient programs.

## Card: Why it matters
Functions are fundamental to organizing and reusing code, making programs cleaner, shorter, and easier to manage. Understanding them is essential to programming beyond the basics, and the underlying idea — package a task once, reuse it many times — is intuitive and applies far beyond coding.

## Card: Key terms
- **Function**: a named, reusable block of code performing a task.
- **Calling a function**: running it by its name.
- **Input (parameters)**: information you give a function.
- **Output (return value)**: what a function gives back.

## Card: Package a task, reuse it
A **function** packages a specific task into a named block. Define it once — say, a function called `addNumbers` that adds two numbers — and then **call** it (run it by name) whenever you need that task done, instead of rewriting the code each time. This is like having a labeled tool you can grab repeatedly. Functions keep programs from being full of repeated code and make them far easier to read and maintain.

## Card: Inputs and outputs
image: inputsOutputsHero
Functions often take **inputs** (called parameters) — information you give them to work with — and produce an **output** (a return value) — a result they give back. For example, `addNumbers(3, 5)` takes the inputs 3 and 5 and returns the output 8. This input-process-output pattern makes functions flexible: the same function can handle different inputs and give appropriate results, like a little machine you feed and get something back from.

## Card: Common mistakes
Common misunderstandings: thinking you must rewrite code for each use (functions let you reuse it); finding functions abstract (they're just packaged, reusable tasks); confusing inputs and outputs; and underrating how much they improve code clarity. Functions are a simple, powerful idea: define a task once, then reuse it as often as needed with different inputs.

## Card: Quick recap
A function is a named, reusable block of code that performs a specific task — defined once and "called" whenever needed, instead of rewriting code. Functions often take inputs (parameters) and give back an output (return value), like a little machine you feed and get a result from. They keep code clean, short, and reusable — a key idea for writing good programs.
